# Deployment Guide

This guide explains how to deploy the PotentialTravelHub Cloud application from scratch using AWS managed services.

---

# Deployment Architecture

The application consists of the following components:

* Amazon S3 (Frontend Hosting)
* Amazon CloudFront (Content Delivery)
* Amazon API Gateway (REST API)
* AWS Lambda (Business Logic)
* Amazon DynamoDB (Inquiry Storage)
* Amazon SNS (Email Notifications)
* Amazon S3 (Document Uploads)
* Amazon CloudWatch (Monitoring)
* AWS IAM (Permissions)

Deploy the infrastructure in the order shown below.

---

# Step 1 – Clone the Repository

Clone the repository:

```bash
git clone https://github.com/Vickusen23/potentialtravelhub-cloud.git
```

Navigate into the project directory:

```bash
cd potentialtravelhub-cloud
```

---

# Step 2 – Deploy the CloudFormation Stack

Navigate to:

AWS Console → CloudFormation

Deploy the CloudFormation template located in:

```text
cloudformation/
```

The template provisions the core backend infrastructure, including:

* DynamoDB Table
* SNS Topic
* IAM Execution Role
* Inquiry Processing Lambda Function

Wait until the stack reaches the **CREATE_COMPLETE** status.

---

# Step 3 – Configure Amazon S3

Create two Amazon S3 buckets.

## Frontend Bucket

Purpose:

* Hosts the HTML, CSS, and JavaScript files.

Example:

```text
potentialtravelhub-frontend
```

Enable:

* Static Website Hosting

Upload:

* index.html
* styles.css
* script.js

---

## Document Bucket

Purpose:

Stores files uploaded by users.

Example:

```text
potentialtravelhub-documents-859462937318-us-east-1
```

This bucket receives uploaded documents using Amazon S3 pre-signed URLs generated by the UploadURLHandler Lambda function.

---

# Step 4 – Deploy Lambda Functions

Deploy the Lambda functions located in:

```text
lambda/
```

Functions include:

* InquiryHandler
* UploadURLHandler

Configure the required environment variables.

Example:

```text
BUCKET_NAME=potentialtravelhub-documents-859462937318-us-east-1
```

Assign the appropriate IAM execution role with permissions to:

* Amazon DynamoDB
* Amazon SNS
* Amazon S3
* Amazon CloudWatch

---

# Step 5 – Configure API Gateway

Create an HTTP API using Amazon API Gateway.

Create the following routes:

```text
POST /inquiry
```

```text
POST /generate-upload-url
```

Attach the corresponding Lambda integration to each route.

Enable Cross-Origin Resource Sharing (CORS).

Recommended settings:

* Allowed Origin: CloudFront URL
* Allowed Methods:

  * POST
  * OPTIONS
* Allowed Headers:

  * Content-Type

Deploy the API.

---

# Step 6 – Configure CloudFront

Create a CloudFront distribution.

Origin:

Frontend Amazon S3 bucket.

Configure:

* Default Root Object

```text
index.html
```

Wait for deployment to complete.

CloudFront provides secure HTTPS access to the application.

---

# Step 7 – (Optional) Configure Route 53

Create a hosted zone for your domain.

Example:

```text
potentialtravelhub.com
```

Create an Alias record pointing to the CloudFront distribution.

---

# Step 8 – Configure ACM

Request a public SSL certificate using AWS Certificate Manager.

Validate the domain.

Associate the certificate with the CloudFront distribution.

This enables secure HTTPS communication.

---

# Step 9 – Upload the Frontend

Upload the following files to the frontend bucket:

* index.html
* styles.css
* script.js

Invalidate the CloudFront cache after every update.

Navigate to:

CloudFront → Invalidations

Create a new invalidation:

```text
/*
```

Wait for completion before testing.

---

# Step 10 – Test the Application

Open the CloudFront URL.

Verify that users can:

* Submit inquiries
* Upload supporting documents
* Receive success confirmation

Confirm that:

* Inquiry data is stored in DynamoDB.
* Uploaded documents appear in the document bucket.
* SNS sends an email notification.
* Lambda logs are visible in CloudWatch.

---

# Deployment Verification Checklist

Verify each of the following:

✅ Frontend loads successfully

✅ API Gateway responds correctly

✅ Lambda executes successfully

✅ Inquiry stored in DynamoDB

✅ Uploaded document stored in Amazon S3

✅ SNS email notification received

✅ CloudWatch logs generated

# Troubleshooting

Common deployment issues include:

* CORS configuration errors
* Missing Lambda integration
* IAM permission issues
* CloudFront cache delays
* S3 bucket policy configuration
* API Gateway deployment not updated

Refer to:

```text
documentation/07-troubleshooting.md
```

for detailed solutions.
